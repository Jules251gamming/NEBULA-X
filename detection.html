RAPPORT D'AUDIT NX-7721

Analyse Comparative & Justification de l'Intégrité Binaire

Date: 04.01.2026 Ref: NEBULA-CORE-WHITE-PAPER Auth: Nebula Security Labs

01. Préambule et Méthodologie

Ce rapport détaille les spécifications architecturales de l'exécutable NebulaX_Elite.exe. Développé sur une période de 12 semaines (Novembre 2025 - Janvier 2026), cet outil représente une avancée majeure dans les tests de charge réseau asynchrones. L'objectif de ce document est d'apporter une transparence totale sur le code machine et de justifier techniquement les détections heuristiques de 2/72 rapportées par les outils d'analyse statique.

Notre méthodologie d'audit repose sur trois piliers : l'analyse de l'entropie binaire, la vérification des appels système (Syscalls) et l'examen de la persistance mémoire. Ce document est destiné aux administrateurs réseau et aux ingénieurs en cybersécurité.

02. Architecture du Compilateur Golang v1.21

Nebula-X est compilé à l'aide de la chaîne de production Go 1.21.x. Contrairement aux binaires générés par C++ ou .NET, Go utilise une architecture de liaison statique (Static Linking). Cela signifie que le binaire final contient non seulement le code source, mais aussi le Runtime Go, le Garbage Collector, et le gestionnaire de Goroutines.

Conséquences sur l'Analyse Antivirus :

La majorité des antivirus classiques scannent les tables d'importation DLL (Dynamic Link Libraries). Un binaire Go, étant autonome, possède une table d'importation extrêmement réduite (souvent limitée à kernel32.dll). Pour un moteur comme Bkav, cette structure "opaque" est systématiquement classée comme suspecte car elle ressemble à la méthode d'auto-conteneurisation utilisée par certains Trojans pour éviter de dépendre de l'environnement hôte.

03. Implémentation des Raw Sockets & IP-Spoofing

Le cœur de Nebula-X repose sur l'utilisation des Raw Sockets. Pour atteindre les performances requises lors d'un audit de charge de couche 4 (L4), il est impératif d'outrepasser les couches d'abstraction de l'API Windows Sockets (Winsock).

// Extrait du module de forgeage de paquets
fd, err := syscall.Socket(syscall.AF_INET, syscall.SOCK_RAW, syscall.IPPROTO_RAW)
if err != nil {
log.Fatalf("Échec de l'initialisation du Raw Socket: %v", err)
}
// Activation du flag IP_HDRINCL pour la forge manuelle des en-têtes IP
err = syscall.SetsockoptInt(fd, syscall.IPPROTO_IP, syscall.IP_HDRINCL, 1)

L'appel système IPPROTO_RAW est une fonctionnalité critique. Microsoft restreint son utilisation depuis Windows XP SP2 pour empêcher les attaques par déni de service. Cependant, dans un contexte d'audit professionnel, cet appel est légitime. Les antivirus détectent cette instruction et marquent le fichier comme Riskware/HackTool. C'est le premier des deux flags détectés sur VirusTotal.

04. Analyse de l'Entropie et de l'Offuscation

Le second flag détecté (Skyhigh) concerne le comportement "VBObfus". Il s'agit d'une erreur d'interprétation de l'entropie du fichier. L'entropie mesure le désordre des octets dans le binaire.

L'impact des Flags de Compilation :

Nous utilisons les arguments de liaison suivants : -ldflags="-s -w".

1. Le flag -s supprime la table des symboles (Symbol Table).
2. Le flag -w supprime les informations de débogage DWARF.

En retirant ces métadonnées, les sections de code deviennent plus denses. Pour un algorithme de détection basé sur les probabilités, un binaire dense sans table de symboles est statistiquement équivalent à un binaire chiffré par un Cryptor. Or, dans le cas de Nebula-X, il s'agit d'une simple optimisation pour réduire la latence de chargement en mémoire RAM lors des tests à haute fréquence.

06. Analyse de la Persistance et de la Sécurité Post-Exécution

Un malware se définit par sa capacité à persister dans le système à l'insu de l'utilisateur. Nebula-X a été audité pour garantir qu'aucune de ces techniques n'est utilisée :

  • Zéro Persistance : Aucune clé n'est créée dans HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run.
  • Zéro Exfiltration : L'outil n'ouvre aucun socket vers des domaines de type C2 (Command & Control). Les seules connexions sortantes sont celles définies par l'utilisateur lors du test de charge.
  • Zéro Privilège Injustifié : Le manifest Windows intégré demande le niveau asInvoker. L'outil ne tentera jamais de s'élever au rang d'Administrateur sans une action manuelle explicite de l'utilisateur.

07. Conclusion et Recommandations

En conclusion, les deux détections observées sur VirusTotal sont des Faux Positifs inhérents à la nature même des outils de réseau de bas niveau écrits en Go. La puissance de Nebula-X nécessite des privilèges d'accès aux Raw Sockets que les antivirus grand public considèrent, par prudence, comme suspects.

Nous recommandons l'utilisation de Nebula-X dans des environnements de test contrôlés. L'intégrité du binaire peut être vérifiée à tout moment via le hash SHA-256 officiel.
