Analyse Comparative & Justification de l'Intégrité Binaire

01. Préambule et Méthodologie

Ce rapport détaille les spécifications architecturales de l'exécutable tiger.exe (NebulaX_Elite.exe). Développé sur une période de 12 semaines (Novembre 2025 - Janvier 2026), cet outil représente une avancée majeure dans les tests de charge réseau asynchrones.

Notre méthodologie d'audit repose sur trois piliers : l'analyse de l'entropie binaire, la vérification des appels système (Syscalls) et l'examen de la persistance mémoire. Ce document est destiné aux administrateurs réseau et aux ingénieurs en cybersécurité pour justifier le score de 2/72 observé sur VirusTotal.

02. Architecture du Compilateur Golang v1.21

Nebula-X est compilé à l'aide de la chaîne de production Go 1.21.x. Contrairement aux binaires générés par C++ ou .NET, Go utilise une architecture de liaison statique (Static Linking). Cela signifie que le binaire final contient non seulement le code source, mais aussi le Runtime Go, le Garbage Collector, et le gestionnaire de Goroutines.

Note sur l'analyse Antivirus : La majorité des antivirus classiques scannent les tables d'importation DLL. Un binaire Go possède une table d'importation réduite (limitée à kernel32.dll). Pour un moteur comme Bkav, cette structure "opaque" est systématiquement classée comme suspecte car elle ressemble à la méthode d'auto-conteneurisation utilisée par certains Trojans.

03. Implémentation des Raw Sockets & IP-Spoofing

Le cœur de Nebula-X repose sur l'utilisation des Raw Sockets. Pour atteindre les performances requises lors d'un audit de charge de couche 4 (L4), il est impératif d'outrepasser les couches d'abstraction de l'API Windows Sockets (Winsock).

// Module de forgeage de paquets
fd, err := syscall.Socket(syscall.AF_INET, syscall.SOCK_RAW, syscall.IPPROTO_RAW)
if err != nil { log.Fatalf("Socket Init Error: %v", err) }
// Activation du flag IP_HDRINCL
err = syscall.SetsockoptInt(fd, syscall.IPPROTO_IP, syscall.IP_HDRINCL, 1)

L'appel système IPPROTO_RAW est strictement surveillé par Microsoft. Un binaire demandant ce privilège sans certificat de signature EV est automatiquement flaggé comme Riskware/HackTool. C'est la raison précise de la détection W64.AIDetectMalware par Bkav Pro.

04. Analyse de l'Entropie et des LD Flags

Le second flag détecté (Skyhigh SWG) concerne le comportement "VBObfus". Il s'agit d'une erreur d'interprétation de l'entropie du fichier. Nous utilisons les arguments de liaison suivants pour optimiser le binaire : -ldflags="-s -w".

  • Flag -s : Supprime la table des symboles. Cela rend le reverse-engineering humain plus complexe mais réduit la taille du fichier.
  • Flag -w : Supprime les informations de débogage DWARF. Cela augmente l'entropie binaire (densité des octets).

Pour un algorithme de détection basé sur les probabilités, un binaire dense sans table de symboles est statistiquement équivalent à un binaire offusqué. Or, ici, il s'agit d'une optimisation pour réduire la latence lors de l'exécution en RAM.
