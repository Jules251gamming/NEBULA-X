01. Résumé de l'analyse

Lors de la soumission du binaire NebulaX.exe aux moteurs de scan multi-sources (VirusTotal), un ratio de détection de 2/72 a été observé. Ce document détaille les raisons techniques de ces détections et prouve l'absence de code malveillant (Malware-free).

02. Architecture des Raw Sockets

Contrairement aux applications standards qui utilisent les bibliothèques système de haut niveau, Nebula-X communique directement avec les Raw Sockets de la couche Transport. Cette méthode est nécessaire pour :

  • La manipulation précise des en-têtes TCP/UDP.
  • Le contournement des limitations de débit imposées par la pile réseau Windows standard.
  • L'injection directe de paquets pour les tests de stress à haute fréquence.

Note : Les antivirus comme Bkav Pro utilisent des modèles d'IA qui classent automatiquement l'accès direct aux sockets comme un comportement typique des "Backdoors", créant ainsi un Faux Positif.

03. Heuristic & Obfuscation Flags

Le deuxième signalement (Skyhigh SWG) mentionne BehavesLike.Win64.VBObfus. Cette détection est liée aux optimisations de compilation Go :

// Commandes de build générant l'alerte :
go build -ldflags="-s -w" -o NebulaX.exe

// Explication :
-s : Retire la table des symboles.
-w : Retire les informations de debug DWARF.

En retirant ces métadonnées pour alléger le binaire, l'entropie du fichier augmente, ce qui trompe les algorithmes d'analyse statique qui pensent que le code est chiffré ou "caché".
