RAPPORT TECHNIQUE NX-7721

Document d'Audit Interne - Version Élite

01. Analyse de la Signature Binaire

Le projet Nebula-X, compilé via l'architecture Go (Golang) v1.21+, présente un binaire statique. Contrairement aux exécutables liés dynamiquement, l'intégralité des dépendances réseau est incluse dans le binaire lui-même. Cette section explique pourquoi l'absence de fichiers DLL externes déclenche des alertes "Generic Malware" chez certains fournisseurs utilisant des analyses de réputation basées sur le cloud.

Observation : Le binaire utilise le package net de Go, qui implémente sa propre pile réseau pour outrepasser les limitations de WinSock2 (ws2_32.dll) dans certains cas de flood UDP.

02. Appels Système et Couche Abstraite (Syscalls)

Nebula-X accède directement aux API de bas niveau de Windows pour garantir un débit de paquets maximal. Les appels système identifiés comme "sensibles" par les moteurs heuristiques incluent :

// Analyse des segments d'appel (Import Table)
syscall.OpenProcess
syscall.SetsockoptInt (SOL_SOCKET, SO_REUSEADDR)
syscall.RawConn.Control (Injection de paquets bruts)

L'utilisation de SO_REUSEADDR et IPPROTO_RAW permet à l'outil d'utiliser plusieurs ports source simultanément pour simuler un trafic distribué. C'est cette fonctionnalité spécifique qui est signalée par Bkav Pro sous le label W64.AIDetectMalware.

03. Justification de l'Entropie et Section de Debug

Un point critique de l'audit concerne le signalement de Skyhigh (SWG) concernant l'offuscation. En compilation Go, l'utilisation des flags -ldflags="-s -w" supprime :

  • Symbol Table : Nécessaire pour le reverse-engineering humain, mais inutile pour l'exécution.
  • DWARF info : Informations de débogage qui alourdissent le fichier de 30% à 40%.

L'analyseur Skyhigh détecte une "haute entropie" dans les sections de données car il ne trouve pas les métadonnées standard de Microsoft Visual Studio, menant à une classification erronée de "Packer" ou "Obfuscator".

05. Protocole de Vérification Utilisateur

Nous recommandons aux administrateurs réseau de vérifier l'empreinte numérique avant déploiement :

PS > Get-FileHash .\NebulaX.exe -Algorithm SHA256
Hash: 79F09831DF13C974778FDD9F475A0D6868D86E9FDC039C1095D29BA5CE080452

Toute modification du binaire (ajout de virus, injection de code tiers) changera radicalement ce hash. Si le hash correspond, le binaire est identique à la version originale certifiée par nos développeurs.
